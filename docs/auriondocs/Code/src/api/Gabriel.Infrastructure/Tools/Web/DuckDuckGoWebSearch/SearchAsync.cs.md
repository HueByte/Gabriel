SearchAsync orchestrates a two-stage DuckDuckGo web search that prioritizes the rich HTML endpoint for full results and gracefully falls back to the lighter endpoint when necessary. It ensures a session, fetches from the HTML endpoint, and parses results; if anomalies are detected or parsing yields no results, it falls back to the Lite endpoint. If both endpoints return a blocking/anomaly page, it resets the session and throws an InvalidOperationException with guidance for alternative web tools and API keys. If neither endpoint yields results, it returns an empty IReadOnlyList<WebSearchResult>.
