Configures chat provider services in the dependency injection container. It always registers a MockChatProvider as the IChatProvider singleton to guarantee a provider exists (useful for UI selection and to prevent runtime failures when no provider is configured). If a Grok configuration section exists and contains at least one active model, Grok is wired up via the Options pattern: the section is bound to GrokOptions, then a series of validations is applied (ApiKey present, BaseUrl is a valid absolute URL, TimeoutSeconds > 0, at most one active default model, and every model must have a non-empty Name and ContextWindowTokens > 0). Startup-time validation is enabled unless the SKIP_DB_INIT environment flag is set to true. The startup timeout for Grok is read from the configuration (defaulting to 900 seconds if not specified). A GrokAuthHandler is registered as a transient DelegatingHandler, and a named HttpClient is created for the Grok chat provider so authentication is handled by the handler rather than DefaultRequestHeaders. HttpClient.Timeout remains governed by the resilience pipeline rather than per-client settings.
